# Complete News/Blog Platform Specification for Cursor AI

## Project Overview
Create a modern, full-featured news/blog platform using Laravel and SQLite, focused on information systems, programming, and technology content. This is a complete content management system with public-facing website and comprehensive admin panel.

---

## PROMPT 1: Project Setup & Database Architecture

```
Create a Laravel 11 blog/news platform with the following specifications:

PROJECT SETUP:
- Initialize Laravel 11 project named "TechNewsHub"
- Configure SQLite database in .env
- Install required dependencies: Laravel Breeze for authentication, intervention/image for image processing, laravel/sanctum for API

DATABASE SCHEMA:
Create migrations for the following tables:

1. users table (extends default):
   - id, name, email, password, role (admin/editor/author), avatar, bio, status (active/inactive), email_verified_at, remember_token, timestamps

2. categories table:
   - id, name, slug, description, parent_id (nullable for subcategories), icon, color_code, meta_title, meta_description, status (active/inactive), display_order, timestamps

3. tags table:
   - id, name, slug, timestamps

4. posts table:
   - id, user_id, category_id, title, slug, excerpt, content (longText), featured_image, image_alt_text, status (draft/published/scheduled/archived), is_featured, is_trending, view_count, published_at, scheduled_at, reading_time (in minutes), meta_title, meta_description, meta_keywords, timestamps, soft deletes

5. post_tag (pivot table):
   - post_id, tag_id

6. comments table:
   - id, post_id, user_id (nullable), parent_id (for nested comments), author_name, author_email, content, status (pending/approved/spam), ip_address, user_agent, timestamps, soft deletes

7. media_library table:
   - id, user_id, file_name, file_path, file_type, file_size, mime_type, alt_text, title, caption, timestamps

8. pages table (for static pages like About, Contact):
   - id, title, slug, content, meta_title, meta_description, status, template, display_order, timestamps

9. newsletters table:
   - id, email, status (subscribed/unsubscribed), verified_at, token, timestamps

10. settings table:
    - id, key, value, group, timestamps

11. post_views table (for analytics):
    - id, post_id, ip_address, user_agent, viewed_at

12. contact_messages table:
    - id, name, email, subject, message, status (new/read/replied), timestamps

Set up proper foreign keys, indexes, and relationships. Use soft deletes where appropriate.
```

## PROMPT 2: Models & Relationships

```
Create Eloquent models for the TechNewsHub platform with the following relationships and features:

USER MODEL:
- HasMany: posts, comments, media
- Attributes: full_name accessor, avatar_url accessor
- Scopes: active, admins, editors, authors
- Methods: isAdmin(), isEditor(), isAuthor()

CATEGORY MODEL:
- BelongsTo: parent (self-referencing)
- HasMany: children (subcategories), posts
- Attributes: url accessor
- Scopes: active, parent categories, ordered
- Method: getPostsCount()

POST MODEL:
- BelongsTo: user, category
- BelongsToMany: tags
- HasMany: comments (approved comments), views
- Attributes: formatted_date, excerpt_limited, reading_time_text, featured_image_url
- Scopes: published, featured, trending, scheduled, byCategory, byTag, byAuthor, recent, popular
- Methods: incrementViewCount(), isPublished(), canBeEditedBy($user)
- Observers: auto-generate slug, calculate reading time on save

TAG MODEL:
- BelongsToMany: posts
- Attributes: url accessor
- Method: getPostsCount()

COMMENT MODEL:
- BelongsTo: post, user, parent
- HasMany: replies (children)
- Scopes: approved, pending, recent, forPost
- Methods: isApproved(), markAsApproved(), markAsSpam()

MEDIA MODEL:
- BelongsTo: user
- Attributes: url accessor, size_human_readable
- Scopes: images, documents, recent

PAGE MODEL:
- Attributes: url accessor
- Scopes: active, ordered

NEWSLETTER MODEL:
- Scopes: subscribed, unsubscribed, verified
- Methods: verify(), unsubscribe()

Add proper mutators, accessors, and query scopes for efficient data retrieval.
```

```nginx
server {
    listen 80;
    server_name technewshub.com www.technewshub.com;
    root /var/www/technewshub/public;

    add_header X-Frame-Options "SAMEORIGIN";
    add_header X-Content-Type-Options "nosniff";

    index index.php;

    charset utf-8;

    # Gzip compression
    gzip on;
    gzip_vary on;
    gzip_min_length 1024;
    gzip_types text/plain text/css text/xml text/javascript 
               application/x-javascript application/xml+rss 
               application/json application/javascript;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location = /favicon.ico { access_log off; log_not_found off; }
    location = /robots.txt  { access_log off; log_not_found off; }

    error_page 404 /index.php;

    location ~ \.php$ {
        fastcgi_pass unix:/var/run/php/php8.2-fpm.sock;
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        include fastcgi_params;
    }

    location ~ /\.(?!well-known).* {
        deny all;
    }

    # Cache static assets
    location ~* \.(jpg|jpeg|png|gif|ico|css|js|svg|woff|woff2|ttf|eot)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}

9. AUTOMATED BACKUPS:

```php
// app/Console/Commands/BackupDatabase.php
public function handle()
{
    $filename = 'backup-' . date('Y-m-d-H-i-s') . '.sqlite';
    $source = database_path('database.sqlite');
    $destination = storage_path('backups/' . $filename);
    
    File::copy($source, $destination);
    
    // Upload to S3 or external storage
    Storage::disk('s3')->put('backups/' . $filename, 
        File::get($destination)
    );
    
    // Keep only last 30 days
    $this->deleteOldBackups();
}
